# 陪诊管理后台 — 面试话术整理

## 亮点一：菜单即路由动态权限体系

> **简历原文**：基于后端菜单树实现"菜单即路由"动态权限体系，前端通过 Vite 的 `import.meta.glob` 自动生成和注册懒加载路由并持久化；刷新时重建路由状态，避免动态路由丢失导致的白屏问题。

**话术**：这个功能的核心是前端不硬编码路由，由后端菜单树驱动路由生成。登录后调接口拿到当前账号的菜单权限树，然后利用 Vite 的 `import.meta.glob` 批量扫描 views 目录下所有组件，得到一个路径到懒加载函数的映射表。然后遍历菜单树，为每个菜单匹配对应的组件，最后通过 `router.addRoute` 动态注册到路由中。路由数据通过 Pinia 持久化到 localStorage。刷新时动态路由会丢失，所以在 `main.js` 中 `app.use(router)` 之前，从 localStorage 读出菜单数据重新走一遍匹配和注册流程，保证路由注册早于路由匹配，就不会白屏了。
这样做的好处是新增页面只要按约定建好文件、后端配好权限，前端就能自动生效了。

## 亮点二：递归菜单组件与状态联动

> **简历原文**：封装递归菜单组件，实现基于后端配置的多级导航动态渲染；通过 Pinia 统一管理菜单高亮与顶部 Tab 状态，实现路由跳转、菜单选中与 Tab 标签的联动。

**话术**：侧边栏菜单数据来自后端，层级不固定，所以我封装了递归菜单组件：这个组件在渲染时，会v-for遍历菜单数据：如果一个菜单项没有子节点，就渲染成一个el-menu-item；如果它包含一个 children 数组，组件则会渲染一个子菜单容器el-sub-menu，并再次调用自身来处理这个 children 数组。这个过程会一直持续下去，直到所有层级的菜单都渲染完毕。

点击菜单时同时触发三个动作：`router.push` 跳转对应页面、将该页追加到顶部 Tab、更新菜单高亮标识。侧边栏中的菜单项通过 `el-menu` 的 `:default-active` 绑定高亮，顶部 Tab 遍历数组渲染并用当前路由匹配选中态。
关闭 Tab 时做了边界处理，关的是当前页就自动跳相邻 Tab，全关了就回首页。

## 亮点三：Vuex 到 Pinia 迁移与时序 Bug 修复

> **简历原文**：将项目状态管理从 Vuex 迁移至 Pinia，并修复持久化插件异步写入引发的路由跳转时序问题，提升状态同步的确定性。

**话术**：这个项目的状态管理原来用的 Vuex，所以我做了一次迁移。迁移本身不复杂，主要是把 `mutations` 和 `actions` 合并为 `actions`，用 `this` 直接改 state，去掉了 `store.commit` 改成直接调方法，持久化插件也从 `vuex-persistedstate` 换成了 `pinia-plugin-persistedstate`。

但迁移完后出了个 Bug：登录成功后无法跳转首页。原来登录后会执行 `router.push('/')`，根路由中的`redirect`拦截器会去读取 localStorage 里的持久化菜单数据来计算该跳哪个页面。Vuex 的持久化是同步写入的，但Pinia 的持久化走的是微任务队列、是异步的，所以这就导致跳转时 localStorage 还是空的，就又跳回登录页了。解决方案是不依赖 localStorage，直接从 Pinia 内存中算好目标路径再跳转，内存赋值是同步的，不受持久化时机影响。

这里的首页是当前账号有权限的第一个菜单项。
