## 🔥 亮点 1：基于后端菜单树的"菜单即路由"动态权限体系

> **简历描述**：基于后端菜单树实现"菜单即路由"权限体系，利用 Vite Glob 特性自动映射组件，结合 Pinia 持久化与防白屏机制，实现新业务模块接入零前端代码成本。

### 技术实现

**核心代码**：`src/store/menu.js` - `dynamicMenuRender` Action

```javascript
dynamicMenuRender(backendMenuList) {
  // 🔥 Vite Glob：编译时扫描所有 .vue 文件，生成路径 → 异步函数映射
  const modules = import.meta.glob('../views/**/**/*.vue')

  const routerSet = (menuList) => {
    menuList.forEach(item => {
      if (!item.children) {
        // 路径约定：/auth/admin → ../views/auth/admin/index.vue
        const url = `../views${item.meta.path}/index.vue`
        // 动态注入懒加载组件
        item.component = modules[url]
      } else {
        routerSet(item.children) // 递归处理子菜单
      }
    })
  }
  routerSet(backendMenuList)
  this.routerList = backendMenuList
}
```

### 面试问答

**Q: 为什么不直接在前端写死路由表？**

> A: 不同角色用户权限不同，后端控制更安全。传统方案需要手动维护 `import` 语句和路由配置，新增页面改动多处；本方案只需创建 `.vue` 文件，自动生效。

**Q: 刷新页面路由丢失怎么办？**

> A: 在 `main.js` 入口读取 `localStorage` 中的持久化数据，重新调用 `dynamicMenuRender` 和 `router.addRoute`，在 `app.mount()` 之前完成路由恢复。

**Q: 为什么 localStorage 存不了组件函数？**

> A: `localStorage` 只能存 JSON 字符串，`() => import()` 是函数无法序列化。所以刷新后必须重新用路径去 `modules` 映射表里匹配一次。

## 🔥 亮点 2：响应式数据可视化仪表板

> **简历描述**：基于 ECharts 构建数据可视化控制台，使用 ResizeObserver API 实现图表自适应容器尺寸变化，保证布局调整时图表平滑缩放。

### 技术实现

**核心代码**：`src/views/dashboard/index.vue`

```javascript
const initEchart = () => {
  const myChart = echarts.init(echartRef.value)
  myChart.setOption(options)

  // 🔥 ResizeObserver：监听容器尺寸变化，自动触发图表重绘
  const observer = new ResizeObserver(() => {
    myChart.resize()
  })
  observer.observe(echartRef.value)
}
```

### 面试问答

**Q: 为什么不用 `window.resize` 事件？**

> A: `window.resize` 只能监听窗口变化，无法感知 DOM 元素（如侧边栏收起导致的容器宽度变化）。`ResizeObserver` 直接监听元素，更精准。

**Q: 数据格式转换踩过什么坑？**

> A: 后端返回的 `order_money` 是字符串，直接用会导致 `toFixed` 报错。需要在数据处理阶段用 `parseFloat()` 转换：
>
> ```javascript
> order_money: parseFloat(item.order_money) || 0
> ```

## 🔥 亮点 3：Axios 请求拦截与 Token 自动管理

> **简历描述**：封装 Axios 实例，实现 Token 白名单机制自动携带认证头，响应拦截器统一处理业务异常与 Token 过期自动登出。

### 技术实现

**核心代码**：`src/utils/request.js`

```javascript
// 请求拦截：Token 白名单 + 自动携带
http.interceptors.request.use((config) => {
  const whiteUrl = ['/login', '/get/code', '/user/authentication']
  const token = localStorage.getItem('pz_token')
  if (token && !whiteUrl.includes(config.url)) {
    config.headers['X-token'] = token
  }
  return config
})

// 响应拦截：统一错误处理 + Token 过期自动登出
http.interceptors.response.use((response) => {
  if (response.data.code === -1) {
    ElMessage.warning(response.data.message)
  }
  if (response.data.code === -2) {
    // Token 过期
    localStorage.removeItem('pz_token')
    localStorage.removeItem('pz_userInfo')
    router.push('/login')
  }
  return response
})
```

### 面试问答

**Q: 为什么需要 Token 白名单？**

> A: 登录、注册等接口在用户获取 Token 之前调用，此时 Token 不存在。白名单避免这些接口携带 `undefined` 的 Token 导致后端报错。

**Q: 响应拦截统一处理有什么好处？**

> A: 业务组件无需关心异常处理逻辑，只需关注正常流程。Token 过期自动跳转登录页，用户体验更好。

## 🔥 亮点 4：递归菜单组件与动态图标渲染

> **简历描述**：设计递归菜单组件支持无限层级嵌套，结合 Vue 动态组件实现图标按需渲染，菜单高亮状态通过 Pinia 全局管理。

### 技术实现

**核心代码**：`src/components/treeMenu.vue`

```vue
<template v-for="(item, index) in menuData" :key="index">
  <!-- 无子菜单：渲染菜单项 -->
  <el-menu-item v-if="!item.children" :index="`${loopIndex}-${item.meta.id}`">
    <!-- 🔥 动态组件渲染图标 -->
    <el-icon><component :is="item.meta.icon"></component></el-icon>
    <span>{{ item.meta.name }}</span>
  </el-menu-item>

  <!-- 有子菜单：渲染子菜单容器 + 递归调用自身 -->
  <el-sub-menu v-else :index="`${index + 1}`">
    <template #title>
      <el-icon><component :is="item.meta.icon"></component></el-icon>
      <span>{{ item.meta.name }}</span>
    </template>
    <!-- 🔥 递归调用自身，传入子菜单数据 -->
    <treeMenu :menuData="item.children" :loopIndex="`${loopIndex}-${item.meta.id}`" />
  </el-sub-menu>
</template>
```

### 面试问答

**Q: 递归组件如何处理层级索引？**

> A: 通过 `loopIndex` prop 传递父级索引，每一层拼接当前 item 的 id，形成 `1-2-3` 这样的唯一标识，用于菜单高亮状态匹配。

**Q: `<component :is="">` 是什么原理？**

> A: Vue 的动态组件，`is` 属性接受组件名或组件对象。这里 `item.meta.icon` 是 Element Plus 的图标组件名（如 `Platform`），运行时动态渲染对应图标。

## 🔥 亮点 5：订单倒计时组件封装

> **简历描述**：封装通用倒计时组件，支持自定义格式化输出，通过 watch 监听 props 变化实现动态更新，onUnmounted 时清除定时器防止内存泄漏。

### 技术实现

**核心代码**：`src/components/counter.vue`

```javascript
// 计算属性：格式化剩余时间
const formater = computed(() => {
  const duration = Math.max(0, Math.floor(diff.value / 1000))
  const h = Math.floor(duration / 3600)
  const m = Math.floor((duration % 3600) / 60)
  const s = duration % 60
  return props.format
    .replace('hh', h.toString().padStart(2, '0'))
    .replace('mm', m.toString().padStart(2, '0'))
    .replace('ss', s.toString().padStart(2, '0'))
})

// 🔥 关键：组件销毁时必须清除定时器
onUnmounted(() => {
  if (timer) clearInterval(timer)
})

// 监听 props 变化，支持异步数据
watch(
  () => props.millisecond,
  () => startTimer()
)
```

### 面试问答

**Q: 为什么要在 onUnmounted 清除定时器？**

> A: 组件销毁后定时器仍在运行会造成内存泄漏，且可能更新已销毁组件的状态导致报错。

**Q: 如何处理异步数据？**

> A: 使用 `watch` 监听 `props.millisecond` 变化。接口数据返回后触发 watcher，重新启动倒计时。

## 🔥 亮点 6：Tab 筛选与 URL 参数同步

> **简历描述**：订单列表支持 Tab 状态筛选，激活状态与 URL 查询参数双向同步，支持从外部页面跳转时自动定位到指定 Tab。

### 技术实现

**核心代码**：`src/pages/order/index.vue`

```javascript
onMounted(() => {
  // 🔥 从 URL 读取初始状态
  const initialActive = route.query.active
  if (initialActive) {
    active.value = String(initialActive)
  } else {
    active.value = '' // 默认"全部"
  }
  // 根据激活状态加载数据
  getOrderPageList(active.value)
})

// Tab 点击时刷新数据
const SelectTab = (item) => {
  getOrderPageList(item.name)
}
```

**使用示例**：

```javascript
// 从"我的"页面跳转，直接定位到"待支付"Tab
router.push('/order?active=1')
```

### 面试问答

**Q: 这个设计有什么好处？**

> A: 1) 用户刷新页面不会丢失筛选状态；2) 支持外部链接直接跳转到指定分类；3) 方便用户分享特定状态的订单列表链接。

## 🔥 亮点 8：表单验证与弹窗复用设计

> **简历描述**：设计新增/编辑复用同一弹窗组件，通过 nextTick 确保 DOM 更新后回显数据，自定义校验器实现手机号格式验证。

### 技术实现

**核心代码**：`src/views/vppz/staff/index.vue`

```javascript
// 🔥 自定义手机号校验器
const validateMobile = (rule, value, callback) => {
  if (value === '') {
    callback(new Error('请输入手机号'))
  } else {
    const reg = /^1(3[0-9]|4[01456879]|5[0-35-9]|6[2567]|7[0-8]|8[0-9]|9[0-35-9])\d{8}$/
    reg.test(value) ? callback() : callback(new Error('手机号格式不对'))
  }
}

// 🔥 新增/编辑复用同一弹窗
const open = (rowData = {}) => {
  dialogVisible.value = true
  nextTick(() => {
    if (rowData) {
      Object.assign(form.value, rowData) // 编辑时回显
    }
  })
}
```

### 面试问答

**Q: 为什么需要 nextTick？**

> A: `dialogVisible = true` 触发弹窗渲染，但 DOM 更新是异步的。`nextTick` 确保表单 DOM 元素存在后再执行数据回显，避免赋值失败。

**Q: 新增和编辑如何区分？**

> A: 通过 `rowData` 参数判断。点击"新增"按钮传入 `null`，点击"编辑"按钮传入当前行数据。

## 🔥 亮点 9：Vuex → Pinia 状态管理重构

> **简历描述**：完成项目状态管理从 Vuex 4 到 Pinia 的迁移，解决持久化插件异步写入导致的路由跳转时序问题。

### 关键改动

| 方面       | Vuex                       | Pinia                             |
| ---------- | -------------------------- | --------------------------------- |
| Store 创建 | `createStore()`            | `createPinia()` + `defineStore()` |
| 状态修改   | `store.commit('mutation')` | `store.action()` 直接调用         |
| 状态访问   | `store.state.module.xxx`   | `store.xxx` 直接访问              |
| 持久化 Key | `Vuex_data`                | `pinia_menu`                      |

### 面试问答

**Q: 迁移过程中遇到什么问题？**

> A: Pinia 持久化插件是异步的，登录后 `router.push('/')` 触发重定向时，`localStorage` 可能还没写入完成，导致读取失败。解决方案是登录成功后直接从内存计算跳转路径，不依赖 `localStorage`。
